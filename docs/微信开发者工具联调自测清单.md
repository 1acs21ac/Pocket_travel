# 微信开发者工具联调自测清单

## 使用前准备

1. 启动后端：`cd backend-api && npm run dev`
2. 启动前端编译：`npm run dev:mp-weixin`
3. 微信开发者工具勾选：
   - 不校验合法域名
   - 不校验 TLS / HTTPS 证书（仅开发环境）

## 核心闭环检查点

### ✅ 1. 微信登录

- 进入探索页后自动触发登录（无 token 或本地过期时间会重登）
- 预期：后端 **`POST /api/v1/auth/login`**（与 `/api/v1/auth/wechat/login` 等价）返回 token
- 预期：本地存储存在 `token`、`openid`、`tokenExpireAt`（封装于 `src/utils/storage.ts`，底层为 `uni.setStorageSync`）

### 命令行冒烟脚本（可选）

在后端目录执行（需已 `npm run dev` 且本机可访问 `127.0.0.1:3000`）：

```bash
chmod +x backend-api/scripts/smoke-integration.sh
./backend-api/scripts/smoke-integration.sh
```

将顺序调用：健康检查、登录、用户资料、豆子、情绪、路线列表、景点列表、链接导入、路线详情、分享创建、公开读取分享、删除路线。

### ✅ 2. 豆子余额

- 登录完成自动请求 `/api/v1/beans/balance`
- 预期：探索页顶部豆子数与后端返回一致

### ✅ 3. 情绪标签

- 自动请求 `/api/v1/mood-map/profile`
- 预期：顶部显示后端返回的首个情绪标签

### ✅ 4. 路线列表

- 自动请求 `/api/v1/routes`
- 预期：瀑布流卡片渲染路线标题、副标题

### ✅ 5. 链接导入

- 点击“导入”按钮，输入链接
- 预期：调用 `/api/v1/routes/import-link`
- 成功：toast 提示成功并跳转路线详情
- 失败：显示后端错误（如豆子不足、解析失败）

### ✅ 6. 路线详情

- 从探索页或「路径」Tab 进入路线详情
- 预期：请求 **`GET /api/v1/routes/:routeId`**，标题、副标题、途经点与后端一致

### ✅ 7. 景点列表

- 在「路径」Tab 调用 **`GET /api/v1/spots`** 并渲染列表；可对单条调用 **`POST /api/v1/spots/:spotId/favorite`**
- 预期：列表与后端一致，收藏成功有 toast

### ✅ 8. 分享功能

- 在「路径」Tab：**`POST /api/v1/share/route`** 生成 `shareCode`，再 **`GET /api/v1/share/:shareCode`**（无 token）校验公开读
- 预期：返回分享码且公开接口可读到 `exist` / `targetId` 等字段

## 常见错误排查

1. **10002 鉴权失败**
   - 检查 token 是否过期，是否自动触发重新登录
2. **网络错误**
   - 检查 `src/config/env.ts` 的 `API_BASE_URL` 是否为 `http://127.0.0.1:3000`
3. **豆子不足**
   - 后端返回 `20001`，前端应 toast 提示用户
4. **数据库异常**
   - 检查 PostgreSQL 是否启动，`backend-api/.env` 配置是否正确
5. **新用户豆子**
   - 后端新用户首次建账默认 **100** 豆（便于本地 import-link 联调）；老账号仍为库中已有余额
