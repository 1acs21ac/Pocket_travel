# 需求文档

## 1. 应用概述

### 1.1 应用名称
口袋旅行搭子

### 1.2 应用描述
面向18~35岁年轻人的情绪疗愈型周末微旅行规划助手。用户通过完成情绪问答构建个人画像,消耗虚拟积分「豆子」,将模糊的心情状态转化为一条有温度、不踩雷的半日至多日游玩路线。游玩过程中通过打卡拍照、分享机位干货获得豆子奖励,形成「探索灵感—生成规划—现场打卡—赚取豆子」的正向循环。

### 1.3 硬件配件
配套实体玩偶,内置NFC芯片。玩偶形象为文旅相关主题,如地方文化美食、地方景点建筑物、或与中国传统中有美好寓意的事物联系在一起,例如上海的葱油饼玩偶、西安的兵马俑玩偶、柚子姐祛霉运玩偶等。用户将手机靠近玩偶NFC感应区,可快速唤起小程序并进入路径页,增强仪式感与品牌辨识度。

## 2. 用户与使用场景

### 2.1 目标用户
18~35岁,周末想出门散心但厌烦做复杂攻略、熬夜查攻略的年轻人

### 2.2 核心使用场景
- 失恋求治愈、工作压力大想透气、考公考研找安静自习地、暧昧期找约会场所、单身想轻社交认识新朋友
- 周五晚上或周六早上打开小程序,设置偏好后生成路线,跟着路线游玩,打卡拍照评论,分享裂变

### 3.2 底部导航栏
固定显示三个Tab:
- 探索:发现灵感、场景化推荐、快速导入
- 路径:路线生成工厂、行程执行管理器
- 我的:个人情绪档案与数字资产中心

### 3.3 探索页

#### 3.3.1 顶部状态栏(优化布局)
采用左右对称布局,信息层级清晰,视觉轻盈。

**第一行(核心信息行):**
- 左侧:定位城市与天气图标及温度,格式为📍 上海 · ☀️ 22℃,点击可手动切换城市
- 右侧:豆子余额胶囊,格式为🫘 100,点击跳转「我的」页豆子明细

**第二行(情绪状态行):**
- 左侧:当前心情图标与文案,如😌 放松中,右侧带下拉箭头,点击弹出浮层可快速调整情绪状态
- 右侧:情绪问答引导入口(仅当未完成问答或超7天未更新时显示),文案为「更新心情」,带呼吸动效

#### 3.3.2 快捷导入条
- 视觉:白色圆角输入框,左侧放大镜图标,右侧「导入」文字按钮
- 底纹文案:「粘贴小红书/抖音链接,一键抄作业...」
- 交互逻辑:点击输入框后,自动切换底部Tab至「路径」页,并随即弹出「导入自媒体链接」半屏弹窗

#### 3.3.3 场景化推荐瀑布流
- 采用双列卡片流布局,系统结合用户当前情绪标签、社交身份、实时天气生成个性化推荐
- 卡片内容结构:
  - 场景标签:系统根据画像自动打的标签(如#暧昧升温中、#考研上岸局、#周末出逃计划)
  - 标题:一句话描述路线亮点
  - 预览图:多图拼接或主景点图
  - 副标题:途径点预告
  - 社交背书:显示「👥 328人去过」
- 点击卡片交互:
  - 跳转至路线预览页(只读模式,不可打卡)
  - 预览页展示完整行程安排,底部常驻按钮为「🎲 采纳灵感,消耗40豆子生成我的路线」
  - 用户点击按钮并确认后,路线才归属到个人「路径」页并扣除豆子

### 3.4 情绪地图问答页

#### 3.4.1 问答流程
- 首次启动小程序时展示问答页,用户可选择「完成问答」或「跳过」
- 若用户选择跳过,可直接进入探索页使用基础功能
- 后续可在我的页重新编辑完整问答
- 采用卡片滑动式界面,轻量有趣
- 共7个问题,覆盖社交身份、情绪状态、旅行节奏、氛围偏好、主题兴趣、旅行目标、热门态度
- 完成后生成情绪地图头像挂件

#### 3.4.2 问答内容

**问题1:你通常和谁一起出门?(单选)**
- 独自一人
- 伴侣/对象
- 带娃(0~12岁)
- 带青少年(13岁以上)
- 朋友/同事
- 陌生人(想认识新朋友)

**问题2:你现在的状态/心情是怎样的?(可多选)**
- 工作/学业压力大,想放松
- 失恋/情绪低落,需要治愈
- 心情很好,想庆祝或奖励自己
- 在家/宿舍待腻了,想透气
- 社交能量爆棚,想组局热闹
- 只想一个人静静,不想被打扰

**问题3:你喜欢的旅行节奏是?(可多选)**
- 特种兵式:一天刷10个点,累但值
- 背包客式:随性自由,走哪算哪
- 舒适慢节奏:睡到自然醒,慢慢逛
- 运动式:徒步/骑行/爬山,动起来才爽
- 省钱省力:预算低,不想太累

**问题4:你对景点的氛围偏好是?(可多选)**
- 热闹有人气(市集、商圈、网红点)
- 安静小众(避开人群,独享空间)
- 出片好看(拍照第一,构图要美)
- 宠物友好(能带毛孩子)
- 亲子友好(有儿童设施或活动)

**问题5:你对以下哪些主题特别感兴趣?(可多选)**
- 自然风光(山/海/湖/森林)
- 历史人文(古迹、博物馆、老街)
- 美食探店(餐厅、小吃、咖啡店)
- 潮流艺术(展览、买手店、涂鸦墙)
- 夜间经济(夜市、酒吧、夜景)
- 运动户外(骑行道、攀岩、露营)

**问题6:这次旅行你希望获得什么?(可多选)**
- 结交新朋友,扩大圈子
- 放松身心,释放压力
- 创造浪漫回忆(约会/纪念日)
- 收获朋友圈/小红书素材
- 探索未知,满足好奇心
- 锻炼身体,出出汗

**问题7:你对热门推荐的态度是?(单选)**
- 跟着榜单走,大家说好就是好
- 参考热门,但更相信小众真实评价
- 完全避开热门,只找冷门宝藏

### 3.5 路线预览页
- 展示完整行程安排(只读模式,不可打卡)
- 显示路线总览信息:总步行/骑行距离、总花费预估、预计结束时间
- 展示途径点序列(按游玩顺序排列的景点列表)
- 底部常驻按钮:「🎲 采纳灵感,消耗40豆子生成我的路线」
- 用户点击按钮并确认后,路线归属到个人「路径」页并扣除豆子
- 页面右上角显示「分享」按钮,点击后生成包含小程序链接的分享图,用户可分享至朋友圈或微信好友

### 3.6 路径页

#### 3.6.1 NFC玩偶唤起功能

**硬件描述:**
- 配套实体玩偶,内置NFC芯片,写入小程序URL Scheme
- 玩偶形象为文旅相关主题,如地方文化美食、地方景点建筑物、或与中国传统中有美好寓意的事物联系在一起,例如上海的葱油饼玩偶、西安的兵马俑玩偶、柚子姐祛霉运玩偶等

**交互流程:**
1. 用户将手机(需开启NFC功能)靠近玩偶NFC感应区域
2. 手机识别NFC信号后,自动拉起微信并打开「口袋旅行搭子」小程序
3. 小程序启动后,全屏展示加载动画:
  - 动画内容:旅行搭子形象(小狗、小猫或行走的葱油饼等)从远处跳跃或行走而来的动态效果
  - 伴随文案:「你的旅行搭子正在向你走来...」
  - 动画时长约1.5秒
4. 动画结束后,直接进入路径页(若用户未登录则先引导登录)
5. 若用户已在路径页有「进行中的路线」,则高亮展示该路线;若无,则展示空态并引导生成路线

#### 3.6.2 页面头部操作区
双按钮入口:
- 🎲 心情生成:点击进入偏好设置表单。若用户未设置情绪标签,则引导至情绪问答页
- 🔗 导入链接:点击弹出半屏弹窗,支持粘贴小红书、抖音等平台链接

#### 3.6.3 进行中行程卡片
- 样式:置顶大卡片,背景使用景点模糊处理图
- 左上角状态标签:胶囊形状,背景色醒目(橙色),文字为「旅行中」
- 标题行:路线名称(如「迪士尼魔法与古镇寻味」)
- 日期与规模:显示「04.22至04.23 · 2天1晚 · 11个地点」
- 进度展示:进度条及文案「已完成 3/6 个地点」
- 底部功能按钮(三个):
  - 问一问:点击唤起AI助手浮窗,可询问路线相关问题(如「下一站怎么走最快?」)
  - 清单:点击展开该路线所需物品清单(如身份证、充电宝、防晒霜),支持勾选
  - 天气:点击显示该路线覆盖日期的天气预报
- 点击卡片主体:直接进入路线详情页(地图+打卡模式)

#### 3.6.4 历史行程列表
- 分组折叠:按月份分组(如「2026年04月」、「更早」),默认展开最近一期,其余折叠
- 卡片样式:
  - 左侧:景点缩略图(圆角)
  - 右侧上部:路线标题(如「西湖区经典必玩2天行程」)
  - 右侧中部:日期与规模(如「04.22至04.23 · 2天1晚 · 17个地点」)
  - 右侧下部:首张打卡缩略图或默认图标
- 点击卡片:进入路线详情页(回顾模式,隐藏打卡按钮,仅展示历史上传内容)

#### 3.6.5 偏好设置表单
- 页面顶部包含「偏好设置」和「导入链接」两个Tab,默认选中「偏好设置」
- 表单区域布局:
  - 行程天数:标题「行程天数」,下方四个选项胶囊:半天、1天、2天、更多。点击「2天」或「更多」时,底部弹出日期选择器,支持选择起止日期。默认选中1天
  - 景点偏好:标题「景点偏好」,下方多选标签网格排列,选项包括:博物馆、公园、商场、寺庙、网红打卡点、咖啡馆、历史建筑、自然风光、夜景。选中状态为填充色,支持多选
  - 游玩时段:标题「游玩时段」,分为「开始时间」(默认10:00)和「结束时间」(默认18:00),右侧带时间选择器入口
  - 氛围倾向(联动场景映射):标题「氛围倾向」,选项内容根据用户当前社交身份+情绪状态动态生成一对反义或互补的选项。联动规则示例:
    - 社交身份=伴侣/暧昧期 & 情绪=想庆祝 → 选项:私密约会风 / 热闹互动风
    - 社交身份=独自一人 & 情绪=压力大 → 选项:完全放空 / 轻度社交
    - 社交身份=朋友/同事 & 情绪=社交能量爆棚 → 选项:打卡出片优先 / 体验互动优先
    - 社交身份=独自一人 & 情绪=只想静静 → 选项:极致安静 / 白噪音陪伴
    - 若用户未完成情绪问答,则默认展示通用选项:节奏舒缓 / 充实丰富
    - 交互形式:两个胶囊按钮,二选一
  - 必去景点(可选):标题「必去景点(可选)」,下方输入框,底纹文案:「输入1-3个你想去的景点名称,用逗号分隔」
- 底部操作栏:
  - 显示「当前豆子:100」及消耗提示「消耗 40 豆子」
  - 按钮:「生成路线」,主色调填充

#### 3.6.6 导入自媒体链接界面
- 页面顶部包含「偏好设置」和「导入链接」两个Tab,默认选中「导入链接」
- 界面布局:
  - 标题:「导入自媒体链接」
  - 说明文案:「支持小红书、抖音、微信公众号等平台链接」(小字灰色)
  - 输入区:「粘贴链接」标签,输入框底纹文案为「https://...」
- 底部操作栏:
  - 显示「当前豆子:100」及消耗提示「消耗 40 豆子」
  - 按钮:「一键导入」,主色调填充

### 3.7 路线详情页

#### 3.7.1 上半屏地图总览

- 展示起点至终点连线,所有途径点标记为图钉
- 图钉语义化颜色:🔴 热门商圈 / 🟢 安静治愈 / 💛 拍照出片
- 顶部悬浮筛选胶囊:文案为「只看适合我心情的景点」,点击后仅显示与当前情绪标签匹配的地点
- 页面右上角显示「分享」按钮,点击后生成包含小程序链接的分享图,用户可分享至朋友圈或微信好友

#### 3.7.2 下半屏途径点卡片流
- 可滑动抽屉
- 日期标题头:显示游玩日期,如「🗓️ 04.22 周三」,右侧可带天气图标
- 路线主题标题:如「迪士尼奇梦之旅」(字体加粗大号)
- 景点卡片结构:
  - 序号与名称:如「1. 上海迪士尼乐园」
  - 简短建议描述:如「建议提前下载官方App预约入园并查看排队时间。开园后直奔热门项目能节省大量时间。」超过三行默认折叠,点击「展开」查看全部
  - 建议游玩时长:文案如「建议游玩时长:一整天」或「建议游玩时长:1小时」
  - 右侧缩略图:景点图片
- 卡片间连接条:
  - 在两个景点卡片之间,插入连接区域
  - 显示内容为距离与交通耗时,如「1.6公里 · 27分钟」或「🚶 1.9公里 · 31分钟」
  - 视觉上使用虚线或向下箭头连接,明确游览顺序
- 推荐地点模块:在卡片流末尾展示备选景点列表(如「七个小矮人矿坑 · 奇梦之光幻影秀 · 花车巡游」),供用户灵活增删行程

### 3.8 景点详情页

#### 3.8.1 基础信息区

## 4. 业务规则与逻辑

### 4.1 豆子系统规则
- 每周一00:00自动发放100豆子
- 豆子累计上限为200,超出部分不再发放
- 生成一次路线消耗40豆子
- 一键导入自媒体链接生成路线消耗40豆子
- 打卡拍照一个景点获得10豆子(每个景点每天限一次)
- 评论一个景点获得10豆子(每个景点每天限一次)
- 更新情绪地图获得5豆子(每天限一次)
- 豆子不足时无法生成路线,需等待周一发放或通过打卡/评论/更新情绪获取
- 扣费保护:若生成失败或链接解析失败,不扣除豆子

### 4.2 情绪地图规则
- 首次启动小程序时展示7个问答,用户可选择「完成问答」或「跳过」
- 若用户跳过,可直接使用基础功能(如直接输入景点、时间等信息生成路线)
- 问答结果存储为用户画像核心字段:社交身份、情绪标签、旅行节奏、氛围偏好、主题兴趣、旅行目标、热门态度
- 用户可在我的页通过两种方式编辑情绪地图:
  - 「完整重测」:重新完成7个问答,更新全部情绪标签
  - 「快速调整」:仅修改当前某一次的情绪状态,无需重新完成全部问答
- 更新情绪地图获得5豆子(每天限一次)
- 若用户超过7天未更新情绪,探索页顶部出现提示气泡:「最近心情变了吗?更新一下推荐更准哦」
- 情绪标签在路线生成时权重高于普通偏好标签

### 4.3 情绪地图与探索页场景化推荐联动规则
- 情绪地图问答结果直接影响探索页场景化推荐内容
- 联动逻辑:
  - 问题1(社交身份)与问题2(情绪状态)组合,触发对应场景卡片:
    - 社交身份=伴侣/暧昧期 & 情绪=想庆祝 → 推荐「暧昧升温卡」
    - 社交身份=朋友/同事 & 天气=晴 & 标签=出片好看 → 推荐「姐妹八卦卡」
    - 社交身份=独自一人 & 情绪=失恋/压力大 → 推荐「沉浸式疗愈卡」
    - 社交身份=陌生人(想认识新朋友) → 推荐「社交破冰卡」
    - 标签=考公/考研 & 偏好=安静 → 推荐「上岸自习卡」
  - 问题3(旅行节奏)影响路线中景点数量与时间安排
  - 问题4(氛围偏好)影响景点筛选(如热闹/安静/出片)
  - 问题5(主题兴趣)影响景点类型推荐
  - 问题6(旅行目标)影响路线整体风格
  - 问题7(热门态度)影响推荐景点的热度权重

### 4.4 标签设置与情绪联动规则
- 若用户已完成情绪地图问答,标签设置页根据情绪标签智能推荐相关景点标签
- 推荐逻辑:
  - 情绪为「失恋疗伤」:推荐自然风光、咖啡馆、历史建筑等
  - 情绪为「压力大想放松」:推荐自然风光、艺术展览、夜生活等
  - 情绪为「心情好想庆祝」:推荐潮牌店、夜生活、艺术展览等
  - 情绪为「社交能量爆棚」:推荐夜生活、潮牌店、咖啡馆等
  - 情绪为「只想一个人静静」:推荐自然风光、历史建筑、咖啡馆等
- 若用户跳过情绪地图问答,标签设置页展示全部景点标签供用户自由选择

### 4.5 路线生成规则
- 基于用户当前定位或手动输入起点
- 根据用户设置的偏好(行程天数、景点偏好、游玩时段、氛围倾向、明确想玩的景点)生成路线
- 系统内置RAG能力,基于用户兴趣标签、情绪标签与行程安排,从知识库中检索相关景点信息
- 调用地图API获取POI及距离矩阵
- 利用大模型结合RAG检索结果,根据用户偏好与情绪标签筛选POI并排序
- 情绪标签影响推荐逻辑:
  - 情绪为「失恋疗伤」:优先推荐自然治愈类景点(植物园、河边步道),避免嘈杂或情侣密集处
  - 情绪为「压力大想放松」:优先推荐冥想空间、茶室、SPA馆
  - 情绪为「心情好想庆祝」:优先推荐浪漫景点、夜景、摩天轮、烛光晚餐
  - 情绪为「社交能量爆棚」:优先推荐可容纳多人、有话题性、能拍照发群的地方(桌游店、轰趴馆、特色小酒馆)
  - 情绪为「只想一个人静静」:优先推荐安静角落、避开人群的景点
- 若用户跳过情绪地图问答,系统基于用户直接输入的景点、时间等信息生成路线,不使用情绪标签增强
- 计算路径保证时间可行
- 生成的路线包含途径点序列及每个景点的详细信息

### 4.6 自媒体链接导入规则
- 用户粘贴自媒体链接后,系统解析链接内容
- 提取景点名称、地址、游玩顺序等信息
- 自动匹配地图POI,补充景点详细信息(开放时间、收费、预约信息等)
- 生成完整路线,包含途径点序列及每个景点的详细信息
- 导入成功消耗40豆子
- 解析失败不扣除豆子,提示用户手动设置

### 4.7 场景化推荐规则
- 系统结合用户当前情绪标签、社交身份、实时天气生成个性化推荐
- 场景映射逻辑:
  - 社交身份=伴侣 & 情绪=想庆祝:推荐暧昧升温卡,必含双人互动体验(陶艺/密室)+ 氛围感餐厅 + 夜景散步点
  - 天气=晴 & 标签=出片好看:推荐姐妹出街卡,必含古着店/买手店 + 网红咖啡厅 + 滨江拍照点
  - 标签=考公/考研 & 偏好=安静:推荐上岸自习卡,必含图书馆预约攻略 + 平价轻食 + 深夜自习室
  - 情绪=失恋 & 标签=治愈:推荐自我疗愈卡,必含林间寺庙/公园 + 高分甜品店 + 猫咖/书店
  - 标签=想认识新朋友:推荐社交破冰卡,必含开放麦/剧本杀 + 青年食堂拼餐 + 草坪音乐会

### 4.8 路线预览与采纳规则
- 用户点击探索页推荐卡片后,跳转至路线预览页(只读模式,不可打卡)
- 预览页展示完整行程安排,底部常驻按钮为「🎲 采纳灵感,消耗40豆子生成我的路线」
- 用户点击按钮并确认后,路线才归属到个人「路径」页并扣除豆子
- 未扣费前不生成个人路线

### 4.9 评论个性化排序规则
- 每条评论关联发布者的标签集合与情绪标签
- 当前用户查看景点时:
  - 先筛选出与当前用户标签交集≥1且情绪标签相同或相似、且带图的评论
  - 按点赞数排序,取前3条作为优先推荐置顶展示
  - 其余评论按点赞数+时间综合排序,放在「更多」中

### 4.10 打卡与评论限制
- 每个景点每天只能打卡一次并获得豆子
- 每个景点每天只能评论一次并获得豆子
- 重复打卡或评论不再获得豆子奖励
- 打卡必须通过GPS校验,距离景点坐标超过500米禁止上传

### 4.11 用户登录与鉴权
- 用户首次进入小程序时,需进行微信授权登录或账号密码登录
- 登录后保存用户信息,用于豆子系统、打卡、评论等功能

### 4.12 天气与人流量信息展示规则
- 探索页顶部展示当天天气情况(温度、天气状态)
- 景点卡片及景点详情页展示当前人流量热力图(颜色等级:畅通/一般/拥挤)
- 景点详情页展示当前天气及温度

### 4.13 RAG智能路线规划规则
- 系统内置旅行知识库,包含景点介绍、旅游攻略、用户评价等信息
- 知识库内容已预先切片并向量化,存储在向量数据库中
- 用户生成路线时,系统根据用户兴趣标签、情绪标签与行程偏好,在知识库中检索最相关的景点信息
- 系统将检索到的知识片段与用户偏好、情绪标签组装成Prompt,调用大模型生成个性化路线规划
- RAG能力在后端运行,用户无需感知,仅体验到更精准的路线推荐

### 4.14 运营冷启动策略
- 官方内容预埋:针对覆盖城市的TOP 50热门景点,通过后台导入官方小编号的高质量带图评论。内容方向涵盖最佳机位(附拍摄参数)、便民攻略(童车租赁、寄存)、周边美食推荐等
- 发布引导示例:在打卡发布页输入框内使用Placeholder展示示例文案(如「比如:进门左转第三个窗户光线最好,适合拍逆光...」),降低用户发布门槛

### 4.15 氛围倾向动态联动规则
- 氛围倾向选项根据用户社交身份与情绪状态动态生成
- 联动规则:
  - 社交身份=伴侣/暧昧期 & 情绪=想庆祝 → 选项:私密约会风 / 热闹互动风
  - 社交身份=独自一人 & 情绪=压力大 → 选项:完全放空 / 轻度社交
  - 社交身份=朋友/同事 & 情绪=社交能量爆棚 → 选项:打卡出片优先 / 体验互动优先
  - 社交身份=独自一人 & 情绪=只想静静 → 选项:极致安静 / 白噪音陪伴
  - 若用户未完成情绪问答,则默认展示通用选项:节奏舒缓 / 充实丰富
- 氛围倾向选择影响路线生成时的景点筛选与排序

### 4.16 分享功能规则
- 用户可在路线预览页、路线详情页、景点详情页点击「分享」按钮
- 系统自动生成包含小程序链接的分享图,分享图包含:
  - 路线分享:路线名称、主要景点预览图、旅行搭子形象、小程序码
  - 景点打卡分享:用户照片、景点名称、旅行搭子形象、小程序码
- 用户可将分享图保存至相册或分享至朋友圈/微信好友
- 其他用户扫描小程序码可直接查看对应的路线或景点详情

### 4.17 NFC玩偶形象规则
- NFC玩偶形象为文旅相关主题,包括但不限于:
  - 地方文化美食:如上海的葱油饼玩偶、北京的烤鸭玩偶等
  - 地方景点建筑物:如西安的兵马俑玩偶、杭州的雷峰塔玩偶等
  - 中国传统美好寓意事物:如柚子姐祛霉运玩偶、招财猫玩偶等
- 小程序内旅行搭子形象可选择小狗、小猫或行走的葱油饼等,与NFC玩偶形象保持风格统一

### 4.18 行程天数选择规则
- 偏好设置表单中行程天数默认选项为:半天、1天、2天、更多
- 点击「2天」或「更多」时,弹出日期选择器,用户可选择起止日期
- 日期选择器支持选择2~7天的行程
- 选择完成后,系统根据起止日期生成对应天数的路线

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户豆子不足40无法生成路线 | 弹窗提示,引导用户打卡/评论赚豆或等待周一发放 |
| 用户跳过情绪地图问答 | 允许使用基础功能,探索页推荐仅基于通用热度,路线生成不使用情绪增强 |
| 用户上传图片失败 | 提示上传失败,允许重试 |
| 生成路线失败(如API调用失败) | 提示生成失败,不扣除豆子,允许重新生成 |
| 用户重复打卡同一景点 | 允许打卡,但不再获得豆子奖励,提示「今日已打卡该景点」 |
| 用户重复评论同一景点 | 允许评论,但不再获得豆子奖励,提示「今日已评论该景点」 |
| 评论文字超过200字 | 输入框限制字数,超出部分无法输入 |
| 用户未登录 | 无法使用打卡、评论、豆子等功能,引导用户登录 |
| 分享图生成失败 | 提示生成失败,允许重新生成 |
| 天气API调用失败 | 不展示天气信息或展示默认提示 |
| 人流量热力图数据获取失败 | 不展示热力图或展示默认提示 |
| 自媒体链接解析失败 | 提示解析失败,不扣除豆子,允许重新输入链接 |
| 自媒体链接内容无法提取景点信息 | 提示无法识别景点信息,不扣除豆子,建议使用偏好设置表单生成路线 |
| 向量数据库检索失败 | 系统基于通用POI数据生成路线,不使用RAG增强 |
| 用户超过7天未更新情绪 | 探索页顶部出现提示气泡:「最近心情变了吗?更新一下推荐更准哦」 |
| 微信授权失败 | 提示授权失败,引导用户重新授权或使用账号密码登录 |
| 小程序码生成失败 | 提示生成失败,允许重新生成 |
| 打卡GPS校验不通过 | Toast提示「你离景点还有点远哦,到达后再打卡吧」,不可上传照片 |
| 探索页点击快捷导入条 | 自动切换底部Tab至「路径」页,并随即弹出「导入自媒体链接」半屏弹窗 |
| NFC触碰时手机未联网 | 正常拉起小程序,但网络请求失败时展示离线提示,引导联网后重试 |
| NFC触碰时小程序未授权 | 进入小程序授权页,授权后进入路径页并展示加载动画 |
| 氛围倾向联动规则无匹配画像 | 回退展示通用选项「节奏舒缓/充实丰富」 |
| 手机未开启NFC | 触碰无反应,用户需手动打开小程序 |
| 小程序未安装 | NFC拉起后进入小程序授权页 |
| 日期选择器选择超过7天 | 限制最多选择7天,超出部分无法选择 |
| 分享图保存失败 | 提示保存失败,允许重新保存 |
| 扫描小程序码失败 | 提示扫码失败,引导用户重新扫描或手动搜索小程序 |

## 6. 验收标准
