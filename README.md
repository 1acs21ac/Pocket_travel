# 口袋旅行搭子 - 微信小程序

## 项目介绍
### 应用定位
「口袋旅行搭子」是面向18~35岁年轻人的**情绪疗愈型周末微旅行规划助手**，核心通过情绪画像构建、AI路线生成、NFC实体交互，打造「探索灵感—生成规划—现场打卡—赚取豆子」的正向闭环，解决用户周末出行攻略繁琐、情绪与场景匹配度低的核心痛点。

### 目标用户
18~35岁，周末想出门散心但厌烦复杂攻略、熬夜查攻略的年轻人群。

### 核心使用场景
- 情绪疗愈：失恋治愈、工作压力释放、独处放松等
- 社交需求：暧昧期约会、轻社交认识新朋友、朋友聚会等
- 效率需求：考公/考研找安静自习地、短途出行快速规划
- 核心行为：周五/周六生成路线 → 游玩打卡 → 分享赚豆

### 核心价值
- 情绪驱动：基于7维度情绪问答生成高匹配度路线，而非单纯景点罗列
- 轻量化：无需复杂攻略，消耗虚拟积分「豆子」即可生成个性化路线
- 实体联动：内置NFC芯片的文旅主题玩偶，快速唤起小程序，强化品牌体验
- 正向激励：打卡/分享赚豆，降低用户创作门槛，提升平台活跃度

## 技术架构
### 整体架构
```
口袋旅行搭子
├── 前端：微信小程序端
├── 后端：服务层 + AI能力层 + 数据层
├── 硬件交互：NFC玩偶适配
└── 第三方服务：地图/天气/支付等API集成
```

### 前端（微信小程序）
- 技术栈：微信小程序原生开发（WXML/WXSS/JS/JSON），可选基于 Taro/uni-app 跨端框架
- 核心能力：
  - 三Tab底部导航栏路由与状态管理
  - NFC模块调用（Android端唤起小程序、页面跳转）
  - 高德/腾讯地图API封装（定位、POI检索、第三方导航调起）
  - GPS定位校验（打卡距离判断）
  - Canvas绘制分享图（含小程序码、行程/景点信息）
  - 本地缓存（用户偏好、情绪标签、豆子余额）

### 后端
- 服务框架：Node.js（Express/NestJS）或 Java（Spring Boot）
- 核心模块：
  - 用户服务：微信授权登录、用户画像（情绪标签/偏好）管理
  - 豆子系统：积分发放/消耗/明细核算（事务级数据一致性）
  - 路线生成服务：RAG检索 + 大模型调用 + 路径规划
  - NFC服务：URL Scheme配置、玩偶唤起逻辑处理
  - 内容服务：评论/打卡/分享内容审核与管理
- 数据层：
  - 关系型数据库：MySQL（用户信息、豆子流水、行程数据）
  - 缓存：Redis（情绪标签、热门景点、临时路线数据）
  - 向量数据库：Milvus/Chroma（RAG知识库向量存储，支撑AI路线生成）
- AI能力层：
  - RAG（检索增强生成）：景点知识库切片+向量化检索
  - 大模型：调用通用大模型（GPT/文心一言）生成个性化路线
  - 推荐算法：基于情绪标签/社交身份/实时天气的场景化推荐

### 硬件交互（NFC玩偶）
- NFC芯片写入微信小程序URL Scheme（`weixin://dl/business/?t=XXXXXX`）
- 适配Android/iOS微信NFC唤起规则（iOS无NFC唤起能力，做兜底引导）
- 小程序端1.5秒加载动画与页面跳转逻辑

### 第三方服务集成
- 地图API：高德/腾讯地图（POI检索、距离矩阵、导航调起）
- 天气API：实时天气/温度获取（支撑场景化推荐）
- 人流量API：景点热力图数据展示
- 微信开放平台：小程序码生成、分享、授权登录

## 核心功能
### 核心页面结构
```
口袋旅行搭子小程序
├── 底部导航栏(三Tab)：探索、路径、我的
├── 探索页：情绪引导 + 快捷导入 + 场景化推荐瀑布流
├── 路线预览页：只读模式行程展示，支持采纳生成个人路线
├── 情绪地图问答页：7维度构建用户情绪/旅行偏好画像
├── 路径页：NFC唤起 + 行程管理 + 偏好设置 + 链接导入
├── 路线详情页：地图总览 + 途径点卡片流
├── 景点详情页：基础信息 + 打卡上传 + 评论互动
├── 我的页：资产管理 + 情绪/标签设置 + 足迹展示
└── 辅助页：标签设置、豆子明细、互动消息、分享图生成
```

### 关键功能模块
1. **情绪地图问答**：7维度（社交身份/情绪状态/旅行节奏等）构建用户画像，支持「快速调整（仅改情绪）」和「完整重测」，每日更新可获5豆子奖励
2. **场景化推荐**：双列瀑布流布局，基于情绪标签/社交身份/实时天气生成个性化卡片（暧昧升温卡、沉浸式疗愈卡、上岸自习卡等）
3. **AI路线生成**：支持「偏好设置」「链接导入」两种方式，基于RAG+大模型生成高匹配度路线，消耗40豆子/次
4. **NFC玩偶唤起**：触碰实体玩偶自动拉起小程序，播放1.5秒加载动画后进入路径页，高亮进行中路线
5. **打卡与互动**：GPS校验打卡（距离≤500米），评论按「同情绪标签+点赞数」个性化排序，打卡/评论各赚10豆子/天/景点
6. **豆子系统**：核心积分体系，支撑「消耗（生成路线）- 激励（打卡/分享）」闭环

## 核心业务规则
### 1. 豆子系统规则
| 行为 | 豆子变动 | 限制 |
|------|----------|------|
| 每周一00:00自动发放 | +100 | 累计上限200，超出不发放 |
| 生成路线/导入链接 | -40 | 不足40无法生成，失败不扣费 |
| 景点打卡 | +10 | 每个景点每天限1次 |
| 景点评论 | +10 | 每个景点每天限1次 |
| 更新情绪地图 | +5 | 每天限1次 |

### 2. 核心规则摘要
- 情绪标签权重高于普通偏好，直接影响探索页推荐、路线生成逻辑
- 打卡需GPS校验（距离>500米禁止上传），重复打卡/评论无豆子奖励
- 分享图包含行程/景点信息+小程序码，扫码可直达对应页面
- 超过7天未更新情绪，探索页顶部提示「更新心情提升推荐精准度」

## 微信小程序开发/部署注意事项
### 1. 权限配置
- 定位权限：申请`scope.userLocation`，用于打卡GPS校验、路线生成起点定位，需在小程序后台配置「位置服务」类目
- NFC权限：仅Android支持，需在`app.json`配置`nfc`模块，iOS端做「手动打开小程序」兜底引导
- 相册权限：申请`scope.writePhotosAlbum`，用于分享图保存至本地
- 网络权限：配置所有后端接口、第三方API（地图/天气）为合法域名，测试阶段可开启「不校验合法域名」

### 2. NFC相关注意事项
- 仅Android微信（版本≥7.0.0）支持NFC唤起小程序，iOS无此能力
- NFC芯片需写入微信小程序URL Scheme（需在微信开放平台配置）
- 处理离线场景：NFC唤起无网络时，正常拉起小程序，展示离线提示并引导联网重试
- 加载动画严格控制在1.5秒内，避免用户等待体验差

### 3. 性能优化
- 瀑布流图片懒加载：探索页推荐卡片图片按需加载，减少首屏加载时间
- 本地缓存策略：情绪标签、豆子余额、最近行程缓存至本地，减少接口请求
- 地图组件优化：路线详情页地图占屏40%，下拉放大时懒加载POI数据
- 分享图生成：使用Canvas离线绘制，避免频繁调用后端生成

### 4. 发布与审核
- 小程序类目：选择「旅游-旅游攻略/行程规划」相关类目，避免审核驳回
- 内容合规：预埋的官方评论需符合微信内容规范，打卡/评论功能需加内容风控
- 版本兼容：适配微信基础库最低版本≥2.24.0，兼容不同机型/系统
- 隐私合规：用户定位、打卡照片等数据需符合《微信小程序隐私保护指引》

### 5. 其他注意事项
- 微信授权登录：处理授权失败场景，提供账号密码登录兜底
- 第三方地图调起：兼容高德/百度/Apple Maps，避免跳转失败
- 豆子系统：后端需做事务处理，避免扣费/发豆数据不一致
- 分享功能：分享图需包含小程序码，内容不得涉及诱导分享

## 异常与边界情况处理
| 场景 | 处理方式 |
|------|----------|
| 豆子不足40生成路线 | 弹窗提示，引导打卡/评论赚豆或等待周一发放 |
| 跳过情绪问答 | 可用基础功能，推荐仅基于通用热度，无情绪增强 |
| 打卡GPS校验不通过 | Toast提示「你离景点还有点远哦，到达后再打卡吧」，禁止上传 |
| 链接解析/路线生成失败 | 不扣豆子，提示「解析失败，请重试或手动生成路线」 |
| NFC触碰无网络 | 拉起小程序，展示「当前网络不佳，请联网后重试」提示 |
| 重复打卡/评论 | 允许操作，无豆子奖励，提示「今日已完成该景点打卡/评论」 |
| 微信授权失败 | 引导重新授权或切换账号密码登录 |
| 分享图生成失败 | 提示「生成失败，请重试」，兜底展示默认分享模板 |

## 快速开始（开发环境）
### 前置条件
- 微信开发者工具（最新版）
- 后端服务已部署，配置好接口域名
- 微信小程序AppID（测试号/正式号）
- Android真机（测试NFC唤起功能）

### 开发步骤
1. 克隆代码仓库：
   ```bash
   git clone [仓库地址]
   cd 口袋旅行搭子
   ```
2. 微信开发者工具导入项目：
   - 选择「小程序」模板，填入AppID
   - 配置项目目录，开启「不校验合法域名、web-view（业务域名）等」（测试阶段）
3. 配置后端接口：
   - 修改`config/api.js`中的接口域名，指向开发环境后端服务
4. 调试核心功能：
   - 模拟器调试基础页面/交互，Android真机调试NFC/定位功能
   - 验证豆子系统、路线生成、打卡等核心流程

## 核心验收标准
1. 底部导航栏固定展示「探索/路径/我的」三Tab，交互正常
2. 情绪问答结果正确关联推荐/路线生成逻辑
3. NFC玩偶可正常唤起小程序并进入路径页，加载动画符合1.5秒要求
4. 豆子系统结算准确，扣费保护生效（生成失败不扣费）
5. 打卡GPS校验、每日次数限制功能生效
6. 分享图生成/扫码跳转功能正常
7. 所有异常场景按规则提示/处理
8. RAG+大模型生成的路线与用户情绪标签高度匹配
